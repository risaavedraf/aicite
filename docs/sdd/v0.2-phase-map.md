# v0.2.0 Milestone — Phase Map

This plan treats **v0.2.0 as a milestone**, not a single implementation phase.
Work is tracked by capability phases for better traceability.

## Preflight (session)

- Mode: interactive
- Artifact store: OpenSpec + Engram
- PR strategy: ask always
- Review budget: 300 changed lines

## Phase breakdown

### Phase 8 — Rename to CITE
Goal: establish final product identity before adding more commands.

Deliverables:
- binary/CLI name migrated to `cite`
- docs command examples updated to `cite`
- local migration checklist (single-user, local)

### Phase 9 — Installation experience
Goal: make install/run paths explicit and reproducible.

Deliverables:
- installation doc refresh (`cargo run`, built binary, release binary)
- install helper script/pattern if needed
- release artifact naming consistency

### Phase 10 — Hierarchical graph foundation
Goal: introduce hierarchy data model and ingestion structure.

Deliverables:
- schema changes (`topics`, `concepts`, `semantic_links`, chunk refs)
- heading-based topic extraction
- sentence-based chunking (30-200 chars)

### Phase 11 — Hierarchical retrieval and CLI
Goal: consume hierarchy in retrieval flow.

Deliverables:
- chunk-first retrieval + topic/concept enrichment
- breadcrumb in context outputs
- `--flat` fallback + scoped query flags

### Phase 12 — Agent UX + verification + release prep
Goal: improve agent consumption and confirm release readiness.

Deliverables:
- prioritized UX flags (`--compact`, `--max-snippet-chars`, filters)
- evaluation fixtures/metrics updated
- docs + release checklist for v0.2.0

## Milestone gate for v0.2.0

Tag v0.2.0 only if all are true:
1. Phase 8-12 acceptance criteria are green
2. test/lint/format pass
3. migration notes are validated in local environment
4. docs match shipped CLI behavior
